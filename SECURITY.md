# Security Policy 🛡️

## Supported Versions

Currently, only the latest `main` branch version is actively supported with security updates. If you are deploying BrixCI internally, please ensure you pull the latest updates regularly.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of BrixCI and its users seriously. If you discover a security vulnerability within BrixCI, please send an e-mail to the core maintainers at **security@brixci.dev** (or open a draft Security Advisory in the GitHub Repository).

**Please do not report security vulnerabilities through public GitHub issues.**

In your report, please include:
- A description of the vulnerability.
- Steps to reproduce it (e.g. malicious payload inside imports).
- Possible impact if exploited.

We will endeavor to respond to your report within 48 hours and patch confirmed vulnerabilities as soon as possible.
